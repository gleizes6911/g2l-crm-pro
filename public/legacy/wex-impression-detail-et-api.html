Inventaire — données WEX utilisées par cette application

Liste des endpoints WEX appelés dans le code (services/wexService.js) et des fichiers JSON locaux. Ce n’est pas le catalogue officiel complet de toute l’API WEX OLS (d’autres routes peuvent exister côté éditeur) : c’est tout ce que notre intégration exploite aujourd’hui.

Authentification

Méthode Endpoint Rôle
POST /login (sur WEX_BASE_URL) Obtention du jeton OAuth (grant_type: password) — mis en cache ~55 min.

Transactions

Méthode Endpoint Données typiques / pagination Persistance locale
POST /transaction/search-transactions Liste des opérations sur plage de dates (transactionList, hasMoreTransactions, totalRows, etc.) ; filtres dans le payload (dates, compte, …). Enrichi puis fusionné dans data/wex/transactions.json (hors git)
POST /transaction/details Détail complet d’une transaction (transactionId) : lieu, cartes, litres, taxes, lignes lineItems, HTML additionalDetails, etc. Champs sélectionnés normalisés dans chaque entrée de transactions.json

Cartes

Méthode Endpoint Rôle Persistance
POST /card/details Liste des cartes du compte (customerNos) ou détail par cardNo (sync complémentaire par PAN). data/wex/cards.json
POST /card/search-cards Fallback si /card/details ne retourne pas de liste exploitable. Idem → cards.json
POST /card/list Fallback secondaire (corps customerNos ou vide selon essais). Idem
POST /card/search Détail carte + indicateurs de contrôle / vélocité pour enrichissement limites. Mergé dans objets carte en mémoire puis cards.json
POST /card/order/default-profile Profil de contrôle par défaut (pour compléter limites si besoin). Non stocké seul ; peut enrichir l’objet carte

Compte

Méthode Endpoint Rôle Persistance
POST /accounts/olssearch Recherche / synthèse compte (plusieurs variantes de corps testées dans le code pour compatibilité API). data/wex/account.json

Fichiers locaux associés (hors appel API direct)

Fichier Contenu
data/wex/sync_meta.json Métadonnées de dernière synchro, erreurs, plage de dates.
data/wex/card_limits.json Limites manuelles / surcharge par numéro de carte (saisie locale).
data/wex/card_vehicle_mapping.json Correspondances manuelles carte ↔ libellé (si WEX ne fournit pas assez).

API Express interne (données déjà synchronisées)

Ces routes ne rappellent pas WEX en direct : elles lisent le cache JSON via jsonStore.

Route Description
GET /api/wex/transactions Transactions paginées (filtre dates / véhicule).
GET /api/wex/stats/summary, /stats/monthly, /stats/vehicles, /stats/drivers Agrégats pour tableaux de bord carburant WEX.
GET /api/wex/account/summary Synthèse compte + consommation mensuelle par carte + limites.
POST /api/wex/sync Lance syncToLocal (appels WEX + écriture locale).
